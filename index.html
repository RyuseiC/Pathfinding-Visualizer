Breadth-First Search (Unweighted)

BFS was originally meant for traversing/searching a tree or graph data structure. It works by checking whether any of the origin's neighbors is the destination. If so, then, the algorithm returns true. If not, then it asks one of its neighbor's the same question to all of the neighbor's neighbors. This cycle is repeated either iteratively or recursively. For the purposes of this site, the algorithm was slightly modified to use what the algorithm learned on its journey in order backtrack its way from the destination to the origin. See Wikipedia page

Dijkstra's Shortest Path First Algorithm (Weighted)

DSPF is very similar to BFS in that it asks the origin's neighbors the question of whether it's the destination or not, and repeats this iteratively or recursively. The difference, however is that DSPF takes into account so-called weights, otherwise known as traffic or anything that would slow someone down on their way to some place. It does this by initializing each node except the origin to have a distance of infinity from the origin. The algorithm also uses an array of nodes sorted by their distance from the origin, which gets updated continuously. The algorithm checks the neighbors of the node closest to the start and updates their distances from the origin only if the path to the neighbor is closer than the array says. That distance would be the number of steps it took to get to the node closest to the start added to the number of steps it takes to get to the neighbor. The second addend will be either 1 or the weight of the weight indicated in the slider. See Wikipedia page
